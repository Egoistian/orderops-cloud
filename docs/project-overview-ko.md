# OrderOps Cloud 프로젝트 소개

## 한 줄 소개

회사별 데이터 격리, 역할 권한, 동시 수정 충돌 처리, 감사 로그를 하나의 흐름으로 연결한 주문 운영 웹 애플리케이션입니다.

## 제품 개요

OrderOps Cloud는 운영 담당자가 여러 주문을 빠르게 탐색하고, 허용된 상태 전이를 수행하며, 변경 이력을 한 화면에서 추적할 수 있도록 구성했습니다. React·Express·PostgreSQL을 기반으로 세션 인증, 회사별 데이터 격리, 역할 기반 상태 전이, 낙관적 동시성 제어, 트랜잭션 기반 감사 로그를 구현했습니다.

보호된 API는 브라우저가 보내는 회사 ID나 역할을 신뢰하지 않습니다. 세션에서 사용자와 회사 컨텍스트를 복원하고, 모든 주문 쿼리를 해당 회사로 제한합니다. 상태 변경 시 서버가 역할과 워크플로 규칙을 다시 검사하며, 화면이 확인한 주문 버전이 오래되었으면 `409 VERSION_CONFLICT`로 덮어쓰기를 차단합니다.

정상 변경은 PostgreSQL 트랜잭션 안에서 주문 버전 증가와 감사 이벤트 기록을 함께 커밋합니다. DB 트리거는 감사 이력의 일반 수정·삭제를 거부합니다. 단위 테스트, PostgreSQL 통합 테스트, Playwright 흐름으로 권한 거부, 회사 격리, 상태 변경·감사 기록, 동시 수정 충돌을 검증합니다.

## 주요 기능

- 회사·역할 선택 기반 로그인과 8시간 세션
- 회사별 주문 지표, 검색, 상태·예외 필터
- 관리자·운영 담당자·열람 전용 권한 분리
- 서버가 계산한 `allowedTransitions` 기반 상태 변경
- 행 잠금과 `expectedVersion`을 결합한 동시 수정 제어
- 주문 변경과 감사 이벤트의 트랜잭션 처리
- 담당자 이름·역할 스냅샷과 append-only 감사 이력
- 공유 환경 보호 모드의 회사 상태 초기화, 고정 감사 메모, 요청 제한
- liveness/readiness 상태 점검과 안정적인 오류 계약
- 반응형 주문 워크스페이스와 키보드 탐색

- React + TypeScript 운영 화면 구현
- Express API 및 세션 인증 구현
- PostgreSQL 테이블·제약·인덱스 설계
- 역할 기반 주문 상태 전이 구현

- 담당자 스냅샷과 append-only 감사 이력 설계
- 낙관적 동시성 제어와 오류 계약 설계

## 기술 스택

`React` · `TypeScript` · `Vite` · `Node.js 24` · `Express 5` · `PostgreSQL 16` · `Playwright` · `Docker Compose` · `GitHub Actions`

## 설계 포인트

- 인증 컨텍스트와 SQL 조건을 함께 적용해 회사별 데이터 경계를 유지합니다.
- 화면 제어와 별개로 서버에서 관리자·담당자·열람 권한을 검증합니다.
- 동시에 수정된 주문을 조용히 덮어쓰지 않고 버전 충돌로 처리합니다.
- 주문 변경과 작업 이력을 하나의 DB 트랜잭션으로 저장합니다.
- 변경 이력의 일반 수정·삭제를 DB 트리거에서 거부하고 당시 담당자 정보를 보존합니다.
- API 계약, 실행 방법, 신뢰 경계, 현재 범위를 문서로 연결했습니다.

## 검토 순서

1. 역할별 계정으로 로그인해 회사와 권한이 세션에 연결되는 흐름을 확인합니다.
2. 열람 전용 계정의 직접 API 호출이 `403`, 다른 회사 주문이 `404`가 되는 통합 테스트를 확인합니다.
3. `expectedVersion`과 `409 VERSION_CONFLICT`가 동시 수정 덮어쓰기를 막는 과정을 확인합니다.
4. 주문 갱신과 감사 이벤트가 한 트랜잭션에서 커밋되는 코드를 확인합니다.
5. 공유 환경 보호 모드의 초기화, 고정 감사 메모, 인스턴스 단위 요청 제한을 확인합니다.

- 요청 ID, 구조화 로그, 오류 추적, 백업 복구 절차
